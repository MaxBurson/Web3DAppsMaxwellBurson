Overview

This is my website for H7006 Web 3D Applications at the University of Sussex. I originally built the medkit, and then liked the idea of survival ideas, like knives, compasses, tents, torches, etc. I finalised my theme as survival items with a Flashlight, Compass, and Medkit. Each model is viewable in a 3D environment allowing you to view the animations, wireframes, etc.

As taught in the labs (and a few extras), the website uses HTML5, CSS3, Bootstrap 5, Three.js, JavaScript, jQuery, AJAX, JSON, PHP, and SQLite.

3D Modelling

Again, as taught in the labs, the three models were built from scratch in Blender, and exported to GLB files ready for the website.

  • Medkit | started as a flat surface, with the corners rounded and walls built up. The lid was built on, but then later separated for the animation. The hinges were added, and the pivot point was moved to here. The red crosses used their own material and were placed on all sides of the object to make it obviously a medkit. The red seemed to work well with an olive green that gave it a military look.
  • Flashlight | next the flashlight was made. Taking basic inspiration from online, it started as a cylinder with a cone shaped end which is where the bulb was eventually added. A 'grip' texture was downloaded from the internet, and pasted onto the handle section of the flashlight. The animation made the bulb, and glass brighten, emitting light. Whilst the switch was slid between positions.
  • Compass | finally, again with some online inspiration, started as a flat disk with the walls built up around it. The needle was a simple cylinder with a dial on it, all with different materials. A compass face texture was downloaded from online, and pasted onto the bottom of the compass, giving it a beautiful finish in combination with the 'brass' like material. The animation was simple and featured the lid closing and opening.

All animations used the keyframing feature in the Blender timeline, and exported with the GLB export, meaning a button can be hooked up through Three.js, so the animations can be viewed.

Design Choices

The website structure was heavily inspired by the website built up in the labs... But I played around with positionings, and colours, to make the website actually keep the 'survival theme' instead of just being a repeat of the lab website. Here are some key choices I made:

  • Dark green navbar | the forest green felt fitting for a survival aesthetic and gives a really nice contrast against the rest of the content.
  • Oswald and Open Sans | I'm a strong believer that even a couple of fonts can really help differ the webpage from a generic one, so I decided on these two fonts as they look nice together.
  • Bootstrap fluid grid | the product cards use col-sm-4 so they are side by side on a computer, but stack nicely on a mobile, without needing further CSS structuring. (The entire website, using Bootstrap, was built with mobile users in mind, keeping it all super mobile friendly in addition to desktop friendly.)
  • Use of textures | The medkit I could not find anywhere that would enhance the look with textures. The flashlight and compass could on the other hand. The compass uses an internet downloaded compass face, and the flashlight uses a grip texture also downloaded from Google. Both bring more realism and a sharper look.

Testing

I ensured to test everything as I built it. The key parts I checked are as follows:

  • Button behaviour | each of the buttons in the 3D website view were tested individually on the different pages, and ensured they performed the task they are supposed to.
  • Responsiveness | checked at different pixel screens using Chrome DevTools, to ensure the website worked perfectly on mobile, to desktop, and any sizes in between.
  • Model checks | each GLB was previewed in gltf-viewer.donmccurdy.com as recommended in the labs, to ensure the objects and animations exported as expected.
  • Cross-browser | ensured that the website worked in Chrome and also Firefox.

A couple of issues, that I managed to fix:

  • Models were showing super dark on the website, this was caused by physicallyCorrectLights being enabled with too little light intensity. I fixed it by removing it and adding a 3-light setup (hemisphere + two directional).
  • Dropdown menu wasn't working... I found out this was caused by Bootstrap JS being loaded 3 times all together. I fixed it by keeping only the one CDN bundle.
  • There was a strange issue with my flashlight model, whilst exporting, wouldn't pick up the emission animation on the bulb and glass properties. A work around for this was manually changing the base colour through JavaScript using Three.js. This work around is commented and highlighted in the script.js.
